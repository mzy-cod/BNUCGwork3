# 第三次实验
## 1. 基础实验
请使用 Python + Taichi 编写程序，可以参考以下逻辑推进：
- 任务 1：初始化与显存预分配
  - 初始化 ti.init(arch=ti.gpu) 以获取最佳性能。
  - 定义常量：屏幕尺寸 800x800，曲线采样数 NUM_SEGMENTS = 1000，最大控制点数 MAX_CONTROL_POINTS = 100。
  - 分配三大 GPU 缓冲区 (ti.Vector.field)：
    1. pixels：尺寸为 800x800，用于存储最终画面。
    2. curve_points_field：尺寸为 1001，用于一次性接收 CPU 算好的曲线坐标。
    3. gui_points：尺寸为 MAX，用于存放 UI 上的控制点（对象池）。
- 任务 2：实现 De Casteljau 算法
  - 编写纯 Python 函数 de_casteljau(points, t)，返回参数t对应的 [x, y] 浮点坐标。
- 任务 3：编写 GPU 绘制内核 (Kernel)
  - 编写一个 @ti.kernel 装饰的函数 draw_curve_kernel(n: ti.i32)。
  - 在这个函数内部，写一个 for i in range(n): 循环。从 curve_points_field 中读取浮点坐标，将其映射为整数像素索引（乘以宽高），并给 pixels 缓冲区对应的位置赋绿色。
  - 提示：被 @ti.kernel 包裹的 for 循环会在 GPU 上极速并行执行，这是解决卡顿的关键。务必在这里进行 0 <= x < 800 的越界检查。
- 任务 4：主循环、曲线逻辑与交互响应
  - 创建 GGUI 窗口 window = ti.ui.Window(...)。
  - 在主循环 while window.running: 中，你需要监听用户的以下交互操作：
    1. 鼠标点击事件：当监听到鼠标左键点击且控制点数量未达到上限时，获取当前鼠标坐标 pos 并加入到控制点列表中。
    2. 键盘清空事件：当监听到键盘按键 c 被按下时，清空控制点列表。
    3. 计算与绘制逻辑：当控制点数量 $$\ge 2$$ 时：
      - 在 CPU 端循环 1000 次，算出所有曲线上的点。
      - 使用 curve_points_field.from_numpy(...) 批量拷贝到 GPU。
      - 调用 draw_curve_kernel(1001) 让 GPU 画图。
      - 将 pixels 传给 canvas.set_image(pixels) 显示。
- 任务 5：绘制交互控制点（对象池技巧）
  - 由于 canvas.circles() 只能接收定长的 Taichi Field。当有 3 个控制点时，如何塞进容量为 100 的 gui_points 里？
  - 技巧：在 Python 中创建一个长度为 100 的 NumPy 数组，全部填充为 -10.0（藏在屏幕外面，GPU 画了也看不见）。然后把真实的 3 个点覆盖到数组的前 3 个位置。最后将其 from_numpy 给 gui_points 并进行绘制。
- 注意事项：
  - 在现代图形学编程中，CPU 和 GPU 是物理分离的，它们之间通过 PCIe 总线通信。如果在 Python（CPU）的 for 循环里，每次算出一点就去修改 Taichi 的 GPU Field（如 pixels[x, y] = 颜色），海量的跨界通信将使帧率大幅降低，造成卡顿。 
  - 正确的做法（Batching）：在 CPU 里把 1000 个点的坐标全算好，装进一个数组，一次性发给 GPU，然后让 GPU 并行地把这 1000 个像素点亮。
  - 此外，现代高性能渲染管线（如 Metal/Vulkan）极其反感在主循环中“动态申请内存”。绘制辅助线时，最好不要传给它长度忽大忽小的数组，而是预先分配一个固定大小的显存池。

## 附加任务1
### 1. 返走样
  - 背景说明：在基础的光栅化绘制中，由于坐标截断（强转为 int）仅点亮单一像素，会导致曲线在视觉上呈现明显的阶梯状边缘，即走样现象。
  - 实验要求：请改进基础的像素渲染逻辑，实现具有平滑边缘的反走样贝塞尔曲线。
  - 算法提示：
    - 数学插值生成的浮点坐标（如 [400.3, 500.8]）具备亚像素级精度。在光栅化时，可考察该精确几何点周围的局部像素邻域（例如 $$3 \times 3$$ 网格），分别计算各像素中心点与该精确浮点坐标之间的空间距离。
    - 基于距离衰减模型，为邻域内的多个像素分配不同的颜色权重或透明度（距离几何点越近，光强贡献越大），通过局部像素集的色彩混合，在视觉上实现平滑过渡的抗锯齿效果。

## 附加任务2
### 2. B 样条曲线
  - 背景说明： 
    - 贝塞尔曲线存在两个明显的局限性：一是全局控制特性（即移动或添加任意一个控制点，整条曲线的形状都会发生改变）；二是阶数绑定（n 个控制点对应 n-1 阶多项式，当控制点较多时，计算复杂度急剧上升且容易发生数值不稳定）。
    - B 样条曲线通过引入节点向量和分段多项式基函数，成功解决了上述问题，实现了“局部控制”（修改一个控制点只影响曲线的一小段），并允许在不增加阶数的情况下无限增加控制点。
  - 实验要求： 在现有交互与渲染管线的基础上，增加对均匀三次 B 样条曲线的绘制支持。建议通过监听特定的键盘按键（例如 b 键），实现在贝塞尔曲线模式与 B 样条曲线模式之间的自由切换，以便直观对比两者的几何行为差异。
  - 算法提示：
    - 数学基础与基函数：B 样条曲线上参数 t 处的点 P(t) 可表示为控制点$$P_i $$与对应的 B 样条基函数$$ N_{i,p}(t) $$的线性组合：$$P(t) = \sum_{i=0}^{n} P_i N_{i,p}(t)$$。对于标准的任意阶 B 样条，可利用 Cox-de Boor 递归公式来计算基函数。
    - 简化实现路径（矩阵形式）：作为图形学入门，可直接针对最常用的均匀三次 B 样条进行分段绘制。每 4 个相邻的控制点构成一段曲线，利用固定的三次 B 样条基矩阵（Basis Matrix）即可通过多项式求值计算出局部坐标，从而避免复杂的递归。
    - 分段拼接逻辑：若有 n 个控制点（$$n \ge 4$$），则曲线由 n-3 段三次曲线平滑拼接而成。在 CPU 端遍历所有分段，分别生成局部采样点，最终汇总至 curve_points_field 并统一发送至 GPU 渲染。
    - 现象观察与验证：在程序中添加超过 4 个控制点，观察贝塞尔曲线与 B 样条曲线的形态差异；尝试在画面局部密集添加控制点，验证 B 样条曲线是否表现出明显的局部控制特性。
